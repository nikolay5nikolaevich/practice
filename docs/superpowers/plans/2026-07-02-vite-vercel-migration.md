# Vite and Vercel Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Convert the standalone HTML report into a minimal Vite project that Vercel can build and publish without changing the page.

**Architecture:** Keep the existing document as Vite's root `index.html`, including its inline CSS and JavaScript. Add only npm metadata and Vite as a development dependency; Vercel will use Vite's standard `build` script and `dist` output automatically.

**Tech Stack:** HTML5, CSS, browser JavaScript, Node.js, npm, Vite 7

## Global Constraints

- Preserve the current page markup, styling, text, and client-side behavior.
- Do not introduce React or another UI framework.
- Do not split the inline CSS or JavaScript into modules.
- Do not add `vercel.json` unless automatic Vite detection fails.

---

## File Structure

- Rename `ui-errors-report.html` to `index.html`: Vite application entry point; contents remain unchanged.
- Create `package.json`: npm metadata, scripts, and the Vite development dependency.
- Create `package-lock.json`: exact dependency resolution generated by npm.
- Create `.gitignore`: excludes installed dependencies and generated build output.

### Task 1: Create and verify the Vite application

**Files:**
- Rename: `ui-errors-report.html` → `index.html`
- Create: `package.json`
- Create: `package-lock.json`
- Create: `.gitignore`
- Generated for verification only: `dist/index.html`

**Interfaces:**
- Consumes: the existing standalone HTML document.
- Produces: `npm run dev`, `npm run build`, and `npm run preview`; Vercel-compatible output in `dist`.

- [ ] **Step 1: Record the original page hash**

Run:

```powershell
(Get-FileHash -Algorithm SHA256 -LiteralPath 'ui-errors-report.html').Hash
```

Expected: a non-empty 64-character SHA-256 value used for comparison after the rename.

- [ ] **Step 2: Rename the Vite entry point**

Run:

```powershell
Move-Item -LiteralPath 'ui-errors-report.html' -Destination 'index.html'
```

Expected: `index.html` exists and `ui-errors-report.html` no longer exists.

- [ ] **Step 3: Verify that the rename did not alter the page**

Run:

```powershell
(Get-FileHash -Algorithm SHA256 -LiteralPath 'index.html').Hash
```

Expected: the hash exactly matches the value from Step 1.

- [ ] **Step 4: Add npm and Vite configuration**

Create `package.json` with exactly:

```json
{
  "name": "ui-errors-report",
  "private": true,
  "version": "1.0.0",
  "type": "module",
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview"
  },
  "devDependencies": {
    "vite": "^7.0.0"
  }
}
```

- [ ] **Step 5: Install the declared dependency**

Create `.gitignore` with exactly:

```gitignore
node_modules/
dist/
```

Then run:

Run:

```powershell
npm install
```

Expected: npm creates `package-lock.json`, installs Vite, and exits with code 0; Git ignores `node_modules` and `dist`.

- [ ] **Step 6: Run the production build**

Run:

```powershell
npm run build
```

Expected: Vite exits with code 0 and writes `dist/index.html`.

- [ ] **Step 7: Verify the production artifact and repository diff**

Run:

```powershell
Test-Path -LiteralPath 'dist/index.html'
git diff --check
git status --short
```

Expected: `Test-Path` prints `True`, `git diff --check` prints no errors, and Git reports the HTML rename plus `.gitignore` and the new npm files without generated files or unrelated changes.

- [ ] **Step 8: Commit the migration**

```powershell
git add -- index.html ui-errors-report.html .gitignore package.json package-lock.json
git commit -m "build: migrate report to Vite"
```

Expected: Git creates one commit containing the Vite migration.
