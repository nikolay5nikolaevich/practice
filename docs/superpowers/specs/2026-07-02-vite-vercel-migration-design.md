# Миграция статического отчёта на Vite для Vercel

## Цель

Преобразовать автономную HTML-страницу в минимальный проект Vite, который собирается локально и автоматически определяется Vercel, не изменяя внешний вид или поведение страницы.

## Текущее состояние

- Проект содержит один файл `ui-errors-report.html`.
- CSS и JavaScript встроены в HTML.
- Внешних ресурсов, зависимостей и серверной логики нет.

## Решение

1. Переименовать `ui-errors-report.html` в `index.html`, потому что Vite использует корневой `index.html` как точку входа.
2. Оставить разметку, стили и клиентский JavaScript без функциональных изменений.
3. Добавить `package.json` с Vite как dev-зависимостью и командами:
   - `npm run dev` — локальная разработка;
   - `npm run build` — production-сборка в `dist`;
   - `npm run preview` — локальная проверка production-сборки.
4. Создать lock-файл через npm для воспроизводимой установки.
5. Не добавлять `vercel.json`: Vercel должен автоматически определить Vite, выполнить `npm run build` и опубликовать `dist`.

## Проверка

- `npm run build` завершается без ошибок.
- В `dist/index.html` присутствует production-страница.
- Исходная разметка и встроенная логика не изменены миграцией.

## Вне области работ

- Перенос на React или другой UI-фреймворк.
- Разделение встроенных CSS и JavaScript на отдельные модули.
- Изменение дизайна, текста или поведения страницы.
- Самостоятельное создание проекта или деплоя в аккаунте Vercel.
